# 全站飘雪背景设计

## 目标

为公开站所有页面添加常驻、低密度、浅色的飘雪背景，营造冬日氛围且不影响阅读、导航或交互。

## 方案

在公开站根应用壳中渲染独立的 `SnowfallOverlay` 组件。组件输出固定数量的装饰性雪花元素，并通过 CSS 自定义属性提供不同的横向起点、尺寸、下落时长和延迟。CSS 使用 `position: fixed`、`pointer-events: none` 和低透明度，使雪花覆盖页面背景而始终位于页面内容之下。

该组件不请求接口、不保存状态，也不增加站内设置或开关。它在所有公开站路由间持续存在，避免每次切换页面重新初始化。

## 响应式与无障碍

- 桌面端保持低密度雪花数量；移动端进一步减少元素数量。
- 雪花仅为装饰，容器使用 `aria-hidden="true"`。
- 当系统启用 `prefers-reduced-motion: reduce` 时，雪花图层自动隐藏。
- 动画仅使用 `transform` 和 `opacity`，避免触发布局计算。

## 样式层级

```
页面背景 / 背景图
  └─ 飘雪图层（固定、不可交互）
       └─ 页面内容、导航与页脚
```

## 验收

1. 首页及所有公开站路由可见缓慢、浅色、低密度的飘雪。
2. 雪花不会遮挡或拦截任何点击、滚动、输入和导航交互。
3. 在移动尺寸下雪花数量较少，页面滚动流畅。
4. 系统启用减少动态效果后，雪花图层不显示。
5. 公开站现有功能和样式不受影响。
