# QQ 音乐悬浮播放器设计

## 目标与范围

为 Nuxt 公开站提供一个仅桌面端显示的悬浮音乐播放器。播放器固定在视口左侧、垂直居中，默认收起，仅露出右侧约 16px 宽的可见边缘；鼠标进入该边缘或已展开的面板时滑出完整面板，鼠标离开后延迟收起。

音乐数据仅来自 QQ 音乐，使用 `@meting/core` 的 Node.js 接口。初始歌单固定为 `9747059163`。不使用第三方公益 Meting HTTP 服务，不提供网易云或其他平台切换，不制作移动版。

## 架构

```text
MusicPlayer.vue（浏览器）
  └─ GET /music/playlist（Nuxt Nitro 同源路由）
       └─ @meting/core：new Meting('tencent')
            └─ QQ 音乐歌单 9747059163
```

`@meting/core` 仅在 Nitro 服务端导入：播放器浏览器代码不会包含该 Node.js 库，也不直接请求 QQ 音乐。服务端将歌单响应转换为公开的前端歌曲模型。标准化曲目中的 `artist` 数组会拼接为显示文本，封面通过 `meting.pic(track.pic_id, size)` 取得，音频通过 `meting.url(track.url_id, bitrate)` 取得；不假定曲目 `id` 可直接作为任一资源查询参数。

QQ 音频地址可能过期，不能把完整歌单的临时 URL 长时间缓存。接口缓存稳定的曲目元数据，当前曲目的播放 URL 仅在请求时按需解析且不做服务端缓存。批量解析时采用有界并发，跳过 URL 为空的 VIP、地区限制或不可播放曲目；`audio` 发生资源错误时，前端重新请求当前曲目的解析地址一次后再展示错误。歌曲时长不由 Meting 标准化数据承诺，播放器以 `audio` 的 `loadedmetadata` 事件获取并维护。

## 服务端接口

新增仅供本站前端使用的读取接口：`GET /music/playlist` 返回歌单元数据，`GET /music/playlist/:trackId/stream` 按需返回该歌单中指定曲目的临时播放地址。由于当前开发代理会匹配 `/api`，本地 Nitro 路由使用不与其冲突的 `/music` 前缀，并在开发与生产环境验证它由 Nitro 而不是 Spring Boot 代理处理。流地址接口仅接受当前已加载歌单中的曲目 ID，避免成为任意第三方资源查询代理。

- 音乐源恒为 `tencent`，歌单 ID 恒为 `9747059163`，通过 `meting.playlist('9747059163')` 读取。
- `Meting#format(true)` 用于取得统一歌曲元数据；验证 QQ 音乐返回的每项含有非空的 `id`、`pic_id` 与 `url_id`，跳过畸形项。歌单数据与可播放 URL 解析异常均返回受控错误。
- 将稳定曲目元数据映射为 `{ id, title, artist, cover }[]`；播放 URL 通过按曲目 ID 的独立读取接口提供，时长由浏览器媒体元数据获得。接口不泄露 Meting 原始结构或服务端配置。
- 元数据缓存命中时直接返回；缓存过期后刷新。若刷新失败且存在过期的旧缓存，可按实现选择安全降级返回旧数据或返回 502。临时播放 URL 不缓存。
- 不在客户端或仓库中保存 QQ 音乐登录 Cookie、Token 或其他凭据。

## 前端播放器

新增一个客户端 Vue 组件，并挂载到 `app.vue`，让其在所有公开页面保持同一播放状态。

### 可见性与交互

- 客户端使用 `matchMedia` 判断桌面断点，并监听其 `change` 事件；只有匹配时才挂载播放器、创建 `audio` 元素和请求歌单。窄屏及 SSR 初始阶段均不挂载播放器、不发起请求、不显示替代入口。
- CSS `position: fixed; left: 0; top: 50%` 固定位置，且设置高于页面内容、低于不应覆盖的全局关键交互层的合适层级。
- 收起态把面板大部分移出左侧，仅保留可悬停的右侧圆角触发条；展开态将完整面板平移至视口内。
- 鼠标移入触发条或面板取消收起计时；移出后使用短延时收起，防止鼠标短暂经过边缘导致闪烁。
- 触发条与按钮提供键盘聚焦与明确的 `aria-label`；键盘聚焦进入面板时保持展开。

### 播放能力

- 使用一个 HTML `audio` 元素维护播放状态。
- 加载歌单后显示首曲但不自动播放，避免浏览器自动播放策略阻止音频。
- 支持播放/暂停、上一首、下一首、点击播放列表切歌、音量、静音、进度显示与拖动跳转。
- 支持顺序播放、单曲循环与列表循环；随机播放不纳入本次范围。
- 音频加载、曲目切换和网络失败均以非阻塞错误提示展示，博客主页面始终可正常使用。

## 视觉与响应式边界

面板延续公开站的毛玻璃风格：使用已有主题变量、半透明背景、圆角、边框与阴影，并自动适配浅色/深色主题。完整面板包含封面、歌名、艺人、播放按钮、进度、音量、切歌控制与可展开的播放列表。

移动端及窄屏不渲染该功能；不做底部抽屉、缩小版播放器或触控等价交互。

## 验证

- 为服务端歌曲映射、固定 QQ 音乐源与缓存分支提供最小自动化测试。
- 为前端播放状态工具提供最小单元测试：切歌边界、播放模式与时间格式化。
- 开发者进行桌面浏览器人工验收：收展、键盘焦点、播放/切歌、错误状态、浅深色主题及窄屏不渲染。
- 运行公开站类型检查与相关测试后再交付。
